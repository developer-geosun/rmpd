# RMPD100 — бизнес-документация

Форма **RMPD100** (Rejestracja przewozu przez przewoźnika) — регистрация нового уведомления о перевозке в реестре SENT для иностранных перевозчиков, выполняющих международные или каботажные перевозки на территории Польши.

> **Источники:** [Услуга PUESC](https://puesc.gov.pl/uslugi/wyslij-zg%C5%82oszenie-przewozu-w-przypadku-miedzynarodowego-przewozu-drogowego-rzeczy-albo-przewozu-kabotazowego-rzeczy-zlozenie-i-aktualizacja-zgloszenia), [FAQ RMPD](https://puesc.gov.pl/faq/-/categories/714841982), спецификация `RMPD_v20.11.2024`

---

## 1. Правовая база и обязанности

### Кто обязан подавать RMPD100

Иностранные перевозчики, выполняющие перевозку на территории Польши при наличии:
- разрешения на международную автоперевозку (zezwolenie),
- разрешения EKMT,
- освобождения от разрешения,
- разрешения на каботаж.

### Сроки вступления обязанности

| Дата | Кого касается |
|------|---------------|
| **01.11.2024** | Перевозчики **без** регистрации в EU/EFTA/Швейцарии |
| **01.01.2025** | Перевозчики **с** регистрацией в EU/EFTA/Швейцарии (перевозки в/из стран вне EU) |

### Ключевые правила

- Подать **до начала** перевозки на территории Польши
- Получить **номер референсный** (reference number)
- Номер действителен **10 дней** с момента выдачи (можно зарегистрировать за 10 дней до старта)
- При изменении данных (кроме товара) — обновить через форму **RMPD** (актуализация)
- Услуга **бесплатна**

## 2. Связанные формы SENT/RMPD

| Форма | Назначение | Когда использовать |
|-------|------------|-------------------|
| **RMPD100** | Регистрация новой перевозки | Новый рейс |
| **RMPD** | Актуализация / дополнение / отмена / статус | Изменение данных существующего уведомления |
| **RMPD406** | Проверка валидности + последняя GPS-позиция | Контроль активности GPS-трекера |

## 3. Данные для заполнения RMPD100

### 3.1. Перевозчик (Przewoźnik)

| Поле | Описание | Примечания |
|------|----------|------------|
| Rodzaj identyfikatora | Тип идентификатора | `INNY` — для не-EU (ЄДРПОУ Украины), `NIP` — Польша, `VAT UE` — EU |
| Numer identyfikatora | Номер ID | Для UA: 8-значный ЄДРПОУ |
| Nazwa | Название компании | **Только латиница** |
| Adres | Адрес регистрации | Kraj, Miejscowość, Kod pocztowy, Ulica, Numer budynku, Numer lokalu |
| Email | Контактный email | Для получения подтверждения |

> Если адрес неизвестен полностью — в пустые поля пишут `BRAK`.

### 3.2. Транспортное средство (Pojazd)

| Поле | Описание | Примечания |
|------|----------|------------|
| Kraj rejestracji | Страна регистрации ТС | Код из словаря |
| Numer rejestracyjny ciągnika | Госномер тягача | Латиница |
| Numer rejestracyjny naczepy | Госномер прицепа | Опционально |
| Numer urządzenia/lokalizatora GPS | ID GPS-устройства | Формат ZSL: `ZXX-XXXXXX-X` или e-TOLL: `MXX-XXXXXX-X` |
| Adres e-mail przewoźnika | Email перевозчика | Для уведомлений |

### 3.3. Разрешение (Informacja o zezwoleniu)

| Поле | Описание |
|------|----------|
| Rodzaj zezwolenia | Тип разрешения (из xlsx-таблицы PUESC) |
| Numer zezwolenia | Номер разрешения |
| Data ważności | Срок действия |

Справочник типов разрешений: [Wykaz wymaganych zezwoleń RMPD](https://puesc.gov.pl/uslugi/wyslij-zg%C5%82oszenie-przewozu-w-przypadku-miedzynarodowego-przewozu-drogowego-rzeczy-albo-przewozu-kabotazowego-rzeczy-zlozenie-i-aktualizacja-zgloszenia) (xlsx PL/EN).

### 3.4. Маршрут и даты (Trasa)

| Поле | Описание |
|------|----------|
| Data rozpoczęcia przewozu | Дата начала на территории PL |
| Data zakończenia przewozu | Дата окончания на территории PL |
| Kraj załadunku | Страна погрузки |
| Kraj rozładunku | Страна выгрузки |
| Miejsca na территории PL | Точки въезда/выезда/транзита |
| Rodzaj przewozu | Тип: с грузом / порожний / транзит / каботаж |

### 3.5. Отправитель и получатель (Nadawca / Odbiorca)

Заполняются **только при перевозке с грузом**:
- Rodzaj identyfikatora, Numer, Nazwa, Adres

### 3.6. Дополнительная информация

| Поле | Описание |
|------|----------|
| Numer dokumentu przewozowego | Номер CMR / транспортного документа |
| Komentarz | Комментарий (опционально) |
| Oświadczenia | Обязательные заявления о полномочиях |

## 4. GPS / геолокализация (SENT-GEO)

Перевозчик обязан передавать геоданные через одно из:

| Способ | Описание |
|--------|----------|
| **e-TOLL PL** | Мобильное приложение, настроенное для SENT |
| **ZSL** | Внешняя система локализации, зарегистрированная в SENT на PUESC |
| **OBU** | Бортовое устройство, зарегистрированное в SENT |

- Один и тот же локатор можно использовать для SENT и RMPD
- Номер локатора из RMPD100 должен совпадать с активным устройством в SENT-GEO
- RMPD406 проверяет валидность уведомления и последнюю GPS-позицию

## 5. XML-спецификация

### Где взять XSD

Скачать архив **RMPD_v20.11.2024** (7z, ~6.66 MB):
https://puesc.gov.pl/uslugi/uslugi-sieciowe-informacje-i-specyfikacje/system-sent

Рекомендуемая структура в проекте:

```
specs/rmpd/
├── RMPD_v20.11.2024/
│   ├── *.xsd          # схемы сообщений
│   ├── examples/      # примеры XML (если есть в архиве)
│   └── README.md      # версия и дата скачивания
```

### Принципы генерации XML

1. Валидировать по XSD **до** подписи
2. Все текстовые поля — **латиница**
3. Коды стран, типов ID — из [словарей PUESC](https://puesc.gov.pl/uslugi/slowniki)
4. Подпись XAdES-BES enveloped (если требует XSD)
5. При расхождении документации и XSD — **приоритет XSD**

### Типы идентификаторов перевозчика

| Код | Применение |
|-----|------------|
| `INNY` | Компании вне EU (Украина — ЄДРПОУ) |
| `NIP` | Польский налоговый номер |
| `VAT UE` | VAT-номер страны EU |

## 6. Результат успешной регистрации

После принятия RMPD100 перевозчик получает:

1. **Номер референсный** (reference number) — в ответе SENT
2. Подтверждение на экране портала
3. Документ в разделе Mój pulpit → Dokumenty
4. Email (если указан в форме)

Номер референсный предъявляется при дорожном контроле вместе с разрешениями.

## 8. Валидация в нашем продукте

### Обязательные проверки перед отправкой

```
□ Все обязательные поля (*) заполнены
□ Только латинские символы в текстовых полях
□ Коды стран/типов из актуальных словарей PUESC
□ Дата начала ≤ дата окончания
□ GPS-номер в формате ZXX-XXXXXX-X или MXX-XXXXXX-X
□ Тип разрешения соответствует маршруту (см. xlsx-таблицу)
□ XSD-валидация пройдена
□ Электронная подпись присутствует (если требуется)
□ Для груженого рейса — заполнены Nadawca/Odbiorca
```

### Условная логика UI

| Условие | Поведение |
|---------|-----------|
| Порожний рейс | Скрыть Nadawca/Odbiorca |
| Тип перевозчика INNY | Показать поле для ЄДРПОУ/рег.номера |
| Каботаж | Показать поля каботажного разрешения |
| Транзит | Упростить маршрутные поля |
